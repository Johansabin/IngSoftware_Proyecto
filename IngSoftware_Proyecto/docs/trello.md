## Estructura del tablero

El tablero combina **User Story Mapping** (columnas por épica funcional) con el flujo **Kanban** de seguimiento (EN PROGRESO → En Revision → COMPLETADO).

### Columnas del tablero

| Columna | Tipo | Descripción |
|---|---|---|
| Product Backlog | Backlog | Todos los requisitos funcionales y no funcionales pendientes |
| SPRINT Actual | Sprint | Historias seleccionadas para el sprint en curso |
| Gestión de Usuarios | Épica | Funcionalidades de registro, login y sesión |
| Seguimiento Emocional | Épica | Registro, historial y visualización de emociones |
| Recomendaciones Emocionales | Épica | Generación de recomendaciones y actividades sugeridas |
| Recordatorios | Épica | Programación y envío de notificaciones |
| Privacidad y Seguridad | Épica | Cifrado, protección de datos y políticas de privacidad |
| EN PROGRESO | Kanban | Tareas actualmente en desarrollo |
| En Revision | Kanban | Tareas completadas pendientes de revisión/prueba |
| COMPLETADO | Kanban | Tareas entregadas y aceptadas |
